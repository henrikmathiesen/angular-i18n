Internationalization (i18n)

The process of changing your software so that it isn't hardwired to one language/locale/culture.

Localization (l10n)

The process of adding the appropriate resources to your software so that a particular language/locale is supported.

Return on Investment

Internationalization, step 1, is a one-time cost (more or less), and so the more languages you localize to, the higher your ROI.

Notes

  1. Run: ng add @angular/localize , to get started.
  2. Also need to make configuration in angular.json. See commit.
  3. Run: ng extract-i18n , this will create a messages file for the sourceLocale
    • XML is default
    • Can use flag for json
    • ng extract-i18n --format json
  4. Duplicate the file, and rename it to match the locale
    • Need to run the command above after marking up for translation
    • And then manually do the translation in the non source locale json file (in this case messages.ja.json)
  5. Make a serve config in angular.json. See commit.
  6. ng serve --configuration ja

Translation is done at build time. So can't really translate text from a backend api(?).
